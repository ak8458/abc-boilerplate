## Part 2: Document-Based Authoring Introduction (30 minutes)

### The New Paradigm

**From This (WordPress):**
- Log into CMS
- Navigate to page
- Click edit
- Use WYSIWYG editor
- Save draft
- Request developer to review/publish

**To This (Edge Delivery):**
- Open document (Word/Google Docs/SharePoint)
- Edit using familiar tools
- Use block tables for components
- Preview in browser
- Publish (no developer needed!)

### Live Demo: Creating a Page

**Demo Steps:**
1. Open document in [selected platform]
2. Add metadata (title, description)
3. Add hero block using table format
4. Add body content (paragraphs, headings, lists)
5. Add cards block
6. Add CTA block
7. Preview in browser
8. Review responsive on mobile/tablet
9. Publish to production

**Block Table Format Example:**

**Hero Block:**
```
| Hero |
|------|
| ![Background image](path/to/image.jpg) |
| # Your Amazing Headline |
| Supporting text that explains the value proposition |
| [Primary Button](link-url) | [Secondary Button](link-url) |
```

**Cards Block:**
```
| Cards |
|-------|
| ![Card 1 image](image1.jpg) |
| ## Card 1 Title |
| Card 1 description text |
| [Learn More](link1) |
| --- |
| ![Card 2 image](image2.jpg) |
| ## Card 2 Title |
| Card 2 description |
| [Learn More](link2) |
```

### Authoring Rules

**Content Structure:**
- Metadata at top of document
- Sections separated by horizontal rules (---)
- Blocks defined using tables
- Default content (paragraphs, headings, lists) between blocks

**Supported Formatting:**
- **Bold text**
- *Italic text*
- Headings (# H1, ## H2, ### H3, etc.)
- Bulleted lists
- Numbered lists
- Links: [Link text](url)
- Images: ![Alt text](image-path)

**Block Tables:**
- First row: Block name
- Following rows: Block content
- Use `---` to separate items within block

## Part 3: Content Workflow Definition (30 minutes)

### Proposed Workflow

**Step 1: Create/Edit**
- Author opens document
- Makes content changes
- Saves (auto-saves)

**Step 2: Preview**
- Author clicks "Preview" (AEM Sidekick extension)
- Preview opens in browser
- Review on desktop/mobile
- Make adjustments as needed

**Step 3: Review/Approve (if required)**
- Author shares document link with approver
- Approver reviews and comments
- Author makes requested changes
- Approver gives approval

**Step 4: Publish**
- Author clicks "Publish" (AEM Sidekick)
- Content goes live instantly (~30 seconds)
- Confirmation message shown

**Step 5: Verify**
- Check live site
- Confirm changes are correct
- Monitor analytics (optional)

## Part 4: Versioning & Rollback (Git Basics) (20 minutes)

### Version Control with Git

**What Authors Need to Know:**
- Every publish creates a commit (save point)
- Full history maintained
- Can view previous versions
- Can rollback to previous version if needed

**Common Scenarios:**

**Scenario 1: View Previous Version**
- Open GitHub repository
- Navigate to page file
- View history
- See who changed what and when

**Scenario 2: Rollback to Previous Version**
- Contact technical team
- Technical team can revert to previous commit
- Re-publish

**Best Practice:**
- Always preview before publishing
- Use meaningful commit messages (auto-generated but can be customized)
- Don't worry - nothing is ever truly lost

## Part 5: Content Governance (20 minutes)

### Content Standards

**Quality Standards:**
- [ ] Spell check before publishing
- [ ] Grammar review
- [ ] Brand voice and tone
- [ ] Accessibility (alt text, headings structure)
- [ ] SEO (meta description, keywords)
- [ ] Links tested and working

**Brand Compliance:**
- Use approved brand assets only

- Use approved messaging

**Content Checklist (Authors):**
- [ ] Content is accurate and up-to-date
- [ ] Spell check passed
- [ ] Images have alt text
- [ ] Links work
- [ ] Meta description added (< 160 characters)
- [ ] Previewed on mobile and desktop

- [ ] Published

### Content Maintenance

**Content Review Cycle:**
- Review content every: ___________ months
- Update outdated content

**Content Ownership:**
- Each page has an owner
- Owner responsible for keeping it current

**Content Archive:**
- Old content moved to archive folder

## Part 6: Images & Assets (15 minutes)

### Working with Images

**Image Sources:**
- Content Hub (preferred - brand-approved assets)
- Upload new images
- Stock photography (with license)

**Image Workflow:**
1. Find or upload image to Content Hub
2. Copy image URL from Content Hub
3. Insert in document: `![Alt text](image-url)`
4. Or: Upload to `/images/` folder in GitHub

**Image Best Practices:**
- File format: JPG (photos), PNG (graphics), SVG (icons/logos)
- File size: Optimize before upload (< 200KB ideal)
- Dimensions: 1920px wide max (responsive)
- File naming: descriptive-kebab-case.jpg
- Alt text: Descriptive for accessibility

**Asset Integration:**
- Content Hub provides approved assets
- Authors browse/search Content Hub
- Download or use URL directly
- Asset metadata auto-applied

## Part 7: Preview & Testing (10 minutes)

### Preview Process

**Types of Preview:**
1. **Local preview:** As you edit (live preview)
2. **Staging preview:** Before publishing
3. **Production preview:** See exactly how it will look live

**What to Check:**
- [ ] Content displays correctly
- [ ] Images load properly
- [ ] Links work
- [ ] Blocks render correctly
- [ ] Responsive (mobile, tablet, desktop)
- [ ] No JavaScript errors (check browser console)

**Tools:**
- Browser DevTools for responsive testing
- AEM Sidekick for preview/publish controls
- PageSpeed Insights (check performance)

**If Something Looks Wrong:**
1. Check block table formatting
2. Validate image URLs
3. Check for typos in links
4. Ask for help in Slack channel
