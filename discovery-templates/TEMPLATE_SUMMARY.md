## Template Usage Guide

### For Project Managers
1. Start with 1.1-project-kickoff-template.md
2. Schedule all sessions based on DISCOVERY_AGENDA.md
3. Customize templates based on ABC's specific context
4. Share pre-read materials 2 days before each session

### For Technical Architects
1. Review 1.3-technical-assessment-template.md before first technical session
2. Use Week 3 and 4 templates for technical deep-dives
3. Customize technical questions based on initial findings
4. Document architecture decisions in real-time

### For DAM Strategists
1. Complete all Week 2 templates
2. Bring taxonomy examples to workshops
3. Prepare to sketch folder structures collaboratively
4. Have metadata schema examples ready

### For Solution Analysts
1. Use business goals template (1.2) to capture requirements
2. Translate business needs into technical requirements
3. Write user stories during discovery
4. Maintain requirement traceability

## Customization Tips

### Before Using Templates

1. **Review RFP and SOW**: Ensure template questions align with ABC's specific requirements

2. **Add ABC Context**: Include ABC-specific terminology, systems, and organizational structure

3. **Adjust for Hybrid Approach**: If discovery sessions are remote/hybrid, add sections for:
   - Virtual collaboration tools
   - Screen sharing logistics
   - Recording permissions

4. **Combine Sessions**: If time is constrained, identify opportunities to combine related sessions

5. **Expand Where Needed**: Add sections for ABC-specific requirements not covered in standard template

### During Discovery

1. **Use Parking Lot**: Capture off-topic items without derailing the session

2. **Document Decisions**: Use Decision Log worksheet for all key decisions

3. **Capture Verbatim**: For critical requirements, capture exact quotes

4. **Take Photos**: Of whiteboard sketches, architecture diagrams, etc.

5. **Note Body Language**: Stakeholder enthusiasm or concern is valuable data

### After Discovery

1. **Complete Within 24 Hours**: Fill in any gaps while fresh

2. **Validate with Participants**: Share notes for confirmation

3. **Link to Other Artifacts**: Connect to user stories, architecture diagrams, etc.

4. **Update Master Documents**: Ensure discovery doc suite is cohesive

## Tools & Resources

### Recommended Tools for Sessions

**Collaborative Whiteboarding:**
- Miro, Mural, or Microsoft Whiteboard
- For architecture diagrams, taxonomy sketches, workflow mapping

**Note Taking:**
- Real-time collaborative docs (Google Docs, Confluence)
- One notetaker per session minimum

**Screen Recording:**
- Record sessions (with permission) for reference
- Especially useful for technical deep-dives

**Screen Sharing:**
- For system demos, current state reviews
- Prepare demo environments in advance

### Reference Materials

**Bring to Sessions:**
- Current website screenshots
- Analytics reports
- User feedback examples
- Competitor examples
- Industry best practices

**Have Available:**
