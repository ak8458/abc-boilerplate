## Part 2: Block Identification & Prioritization (90 minutes)

### Block Discovery Process

For each potential block:
1. **Block Name**: Descriptive, lowercase-hyphen format
2. **Purpose**: What it does
3. **Content Model**: Author inputs (table structure)
4. **Variations**: Different configurations/styles
5. **Used On**: Which pages/sites
6. **Priority**: P0 (MVP), P1 (Phase 2), P2 (Future)
7. **Complexity**: Low/Medium/High
8. **Dependencies**: Other blocks or features needed

---

### Block 1: Header (Global Navigation)

**Priority:** P0 - Required for all sites  
**Complexity:** High  
**Used On:** All 4 websites (780 pages)

**Purpose:**  
Global site navigation with logo, main menu, utility links, search

**Content Model (Authoring Structure):**
```
| Header |
|--------|
| Logo: [Link to homepage] |
| Menu Item 1 | [URL] |
| Menu Item 2 | [URL] |
| -- Submenu Item 2a | [URL] |
| -- Submenu Item 2b | [URL] |
| Menu Item 3 | [URL] |
| Utility: Search | Login | Contact |
```

**Variations:**
- Default (white background)
- Transparent (homepage hero)
- Sticky on scroll
- Mobile: Hamburger menu

**Responsive Behavior:**
- Desktop: Full horizontal menu
- Tablet: Condensed menu
- Mobile: Hamburger with slide-out

**Accessibility:**
- Keyboard navigable
- ARIA labels for menu
- Focus management

**Dependencies:**
- Search functionality (if includes search)
- Login integration (if includes login)

**Development Notes:**
- May need mega-menu for complex navigation
- Ensure mobile performance (no layout shift)
- Sticky header optional based on user feedback

### Block 3: Hero

**Priority:** P0 - Required for MVP  
**Complexity:** Medium  
**Used On:** Homepage, landing pages (~50+ pages)

**Purpose:**  
Large banner/hero section with headline, supporting text, CTA, background image/video

**Content Model:**
```
| Hero |
|------|
| ![Background Image](image.jpg) |
| # Main Headline |
| Supporting text paragraph that explains the value proposition |
| [Primary CTA](link-url) | [Secondary CTA](link-url) |
| Style: centered / left-aligned / dark / light |
```

**Variations:**
- **Style 1:** Centered text, image background
- **Style 2:** Left-aligned text, image right
- **Style 3:** Full-width image, overlay text
- **Style 4:** Video background
- **Style 5:** Carousel/slideshow (multiple heroes)

**Content Inputs:**
- Background image (optimized for web)
- Headline (H1)
- Subheading or body text
- 1-2 CTAs (buttons)
- Alignment preference
- Color overlay (optional)

**Responsive:**
- Desktop: Full height or fixed height
- Mobile: Reduced height, stacked content

**Accessibility:**
- Sufficient color contrast on text over image
- Alt text for background image
- Focus order for CTAs

**Development Considerations:**
- Lazy load background images below fold
- Video background performance optimization
- Ensure text is always readable (overlay if needed)

## Part 3: Template Definition (30 minutes)

### Single Template Structure

**HTML Structure:**
```html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Page Title</title>
  <meta name="description" content="Page description for SEO">
  
  <!-- Open Graph -->
  <meta property="og:title" content="Page Title">
  <meta property="og:description" content="Description">
  <meta property="og:image" content="image-url">
  
  <!-- Styles -->
  <link rel="stylesheet" href="/styles/styles.css">
  
  <!-- Scripts -->
  <script type="module" src="/scripts/scripts.js"></script>
</head>
<body>
  <header class="header"></header>
  
  <main>
    <!-- Blocks/sections rendered here -->
  </main>
  
  <footer class="footer"></footer>
</body>
</html>
```

**Template Features:**
- Responsive meta viewport
- SEO meta tags (title, description)
- Open Graph tags for social sharing
- Schema.org structured data (per page type)
- Global styles loaded
- Block-specific styles lazy-loaded
- JavaScript modules

**Metadata (per page):**
- Title
- Description
- Keywords (optional)
- Author
- Publish date
- Modified date
- Image (for social sharing)
- Type (article, webpage, etc.)

## Part 4: Responsive Design Requirements (30 minutes)

### Breakpoints

**Standard Breakpoints:**
```css
/* Mobile first approach */
/* Mobile: Default (< 768px) */
/* Tablet: 768px and up */
@media (min-width: 768px) { }

/* Desktop: 1024px and up */
@media (min-width: 1024px) { }

/* Wide: 1440px and up */
@media (min-width: 1440px) { }
```

**Content Width:**
- Max content width: 1200px (centered)
- Full-bleed sections: 100% width (hero, CTA)
- Padding: 20px (mobile), 40px (tablet+)

### Device Testing

**Required Testing:**
- iPhone (Safari Mobile)
- Android (Chrome Mobile)
- iPad (Safari)
- Desktop (Chrome, Firefox, Safari, Edge)

**Orientation:**
- Portrait and landscape (mobile/tablet)
