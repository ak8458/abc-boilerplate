## Part 4: Content Conversion Strategy (30 minutes)

### Automated Conversion (Page-Import Skill)

**How It Works:**
1. Provide URL of page to migrate
2. Skill scrapes the page
3. Analyzes content structure
4. Maps to available blocks
5. Generates document with block tables
6. Human reviews and adjusts
7. Publish

**Best For:**
- Standard page layouts
- Repetitive content (blog posts)
- Simple structures
- High volume pages

**Example:** Blog posts (300 pages)
- Automated conversion: 90% done automatically
- Manual review: 10% touch-up
- Estimated: 1 page in 15-20 minutes vs. 1-2 hours manual

### Manual Conversion

**When Needed:**
- Complex custom layouts
- Unique page designs
- High-visibility pages (homepage)
- Pages with special functionality

**Process:**
1. Screenshot existing page
2. Create new document
3. Add metadata
4. Recreate layout using blocks
5. Add content
6. Preview and compare
7. Adjust until matching
8. Publish

**Best For:**
- Homepage
- Key landing pages
- Custom designs
- First pages (learning)

### Hybrid Approach (Recommended)

**Phase 1 (MVP):** 100% Manual
- Learn the process
- Validate blocks
- Build confidence
- Only 5 pages

**Phase 2 (Core):** 80% Manual, 20% Automated
- Important pages done manually
- Start using automation for simpler pages

**Phase 3-4 (Bulk):** 30% Manual, 70% Automated
- Bulk conversion with automation
- Manual review and quality check
- Efficiency at scale

## Part 5: Testing & Validation Strategy (30 minutes)

### Quality Assurance Approach

**Level 1: Developer Testing** (During Migration)
- Block functionality works
- Responsive on all devices
- No JavaScript errors
- Lighthouse score 90+
- Accessibility check (automated)

**Level 2: Content QA** (ABC Team)
- Content accuracy (compare to original)
- Links work correctly
- Images display properly
- Forms function
- Brand compliance

**Level 3: UAT (User Acceptance Testing)**
- Business stakeholders review
- Real user scenarios
- Sign-off before go-live
- ABC owns UAT (per SOW)

**Level 4: SEO Validation**
- Meta tags correct
- URL structure preserved
- Rankings monitored post-launch
- Sitemap updated

### Testing Per Phase

**Phase 1 (MVP - 5 pages):**
- Intensive testing (prove everything works)
- All 4 levels of QA
- Every page tested thoroughly
- Duration: 3-5 days

**Phase 2-4 (Bulk - 775 pages):**
- Automated testing where possible
- Spot check sampling (10-20% of pages)
- Focus on high-traffic pages
- Continuous testing during migration

### Test Checklist (Per Page)

**Visual/Functional:**
- [ ] Layout matches original (or approved design)
- [ ] All content present and accurate
- [ ] Images load and display correctly
- [ ] All links work (no 404s)
- [ ] Forms submit successfully
- [ ] Responsive (mobile, tablet, desktop)
- [ ] Browser tested (Chrome, Firefox, Safari, Edge)

**Performance:**
- [ ] PageSpeed score 90+ (desktop and mobile)
- [ ] Page load < 2 seconds
- [ ] Images optimized
- [ ] No render-blocking resources

**Accessibility:**
- [ ] Lighthouse accessibility score 100
- [ ] Keyboard navigation works
- [ ] Screen reader compatible
- [ ] Color contrast sufficient
- [ ] Alt text on all images

**SEO:**
- [ ] Title tag present and optimized
- [ ] Meta description present (< 160 chars)

- [ ] URL structure preserved (or 301 redirect)
- [ ] Schema markup (if needed)

## Part 6: URL Strategy & Redirects (20 minutes)

### URL Preservation

**Strategy: Keep Existing URLs**
- Maintain current URL structure where possible
- Benefits SEO (preserve rankings)
- No user disruption (bookmarks work)
- No redirect overhead

**Example:**
- Current: `www.ab.bluecross.ca/products/individual/extended-health`
- New: `www.ab.bluecross.ca/products/individual/extended-health` (same!)

### Redirects (If URL Changes)

**When Redirects Needed:**
- URL structure optimization
- Content consolidation (2 pages become 1)
- Page deletion (redirect to relevant page)

**Redirect Approach:**
- 301 Permanent Redirect (for SEO)
- Configured in Edge Delivery metadata
- Format:
  ```
  old-url → new-url (301)
  ```

**Redirect Testing:**
- [ ] All old URLs redirect correctly
- [ ] Redirect is 301 (permanent)
- [ ] No redirect chains (A → B → C)
- [ ] No redirect loops

### Sitemap

**XML Sitemap:**
- Auto-generated by Edge Delivery
- Includes all published pages
- Updated automatically on publish
- Submit to Google Search Console
